Neste artigo veremos rapidamente como se cria um projeto, quais são suas opções e para que serve cada uma delas.

Basicamente não há nenhum segredo, mas como prometido tentarei cobrir a maior parte de informações possíveis sobre as funções do Redmine. Então é isso vamos começar!


Trabalharemos com o passo a passo tradicional e quando necessário colocarei mais informações sobre os pontos mais relevantes.

Acessando a tela de cadastro

1. Primeiro acesse o sistema com uma conta que tenha permissão de administrador

2. No topo da dela acesse a opção administrador e depois a opção 'Projetos', que fica no menu à esquerda da tela;

Exemplo de acesso a tela cadastro de projetos no Redmine

3. Agora estamos na tela de gerenciamento de projetos. Como não temos nenhum criado ainda vamos usar a opção 'Novo Projeto';

Tela de cadastro de projeto com seus campos separados em três seções
4. Agora vamos preencher cada uma de nossas opções:

Opções da seção 1


  1. Campo Nome: Aqui colocamos o nome do nosso projeto;

  2. Campo Descrição : Aqui colocamos uma breve descrição que nos ajude a entender e distinguir o projeto e seus objetivos;

  3. Campo Identificador: Este será um identificador único para identificação do sistema nas funcionalidades do Redmine Por exemplo, ele sera utilizado com um parâmetro na construção da url de acesso ao projeto conforme mostra a figura abaixo. Entao tenha cuidado com  suas siglas ok!
    Exemplo de onde vai o identificador do projeto dentro da url de acesso
  4. Campo opção Público: Esta opção serve para mostrar ou ocultar as informações deste projeto. Quando um projeto é marcado como público as informações gerais deste ficarão disponíveis a qualquer pessoa que acessar o Redmine, estando ela autenticada ou não no sistema. Caso não público somente as pessoas vinculadas ao projeto e os administradores terão acesso a ele.

  5. Campo opção Herdar Membros: essa é uma opção bacana, mas aqui ela esta apresentada como um erro, inclusive vou abrir uma tarefa no site da comunidade para resolver isso. O problema aqui e que este e o primeiro projeto de todo o sistema então não a nada para herdar. Mas após a criação do primeiro projeto antes dessa opção vai existi a opção: subprojeto, veja abaixo como estas duas opções trabalham juntas.

  6. Campo subprojeto de: Essa opção permite vincular o projeto atual em cadastramento como um subprojeto já existente, assim em um exemplo seria possível agrupar sobre um mesmo item todos os projetos de uma empresa.

    Agora sim,  a opção Herdar Membros fará mais sentido. Suponha que em um projeto A, já criado no Redmine, existam trinta pessoas (membros) cadastradas/vinculadas com seus devidos papéis. Assim se durante o cadastro do novo projeto B o colocar como subprojeto do projeto A e marcar a opção 'Herdar', todas estas trinta pessoas  serão automaticamente vinculadas a este novo projeto B. Bacana né poupa um trabalhão podem acreditar.

Opções da Seção 2 e 3

Na seção dois da tela podemos selecionar os módulos que são opções padrões para cada projeto dentro do Redmine. Na seção 3 selecionamos os tipos de tarefas que este projeto poderá utilizar. Por exemplo se este for um projeto de designer, não terá sentido colocar tarefas de desenvolvimento nele. Isso e legal pois permite criar configurações únicas para cada projeto, sem que uma as tarefas de um projeto A interfira em tarefas de um outro projeto B.

Não falei muito sobre as opções da seção 2 e 3 porque mais a frente vamos ter um artigo apenas para falar sobre cada uma delas.

As opções da tela

Pronto! agora existem as duas opções da tela cadastro, são elas:
  •  O botão  'Criar' e o...
  •  Botão 'Criar e continuar'
Podem parecer bobas mas elas são bem uteis, veja como funcionam:

  • Usar a opção 'Criar' significa que ao salvar o projeto você já terminou de cadastrar os projetos que deseja e agora quer personalizar as opções dele. Por isso ao usar esta opção o Redmine redireciona o usuário até a página de personalização do novo projeto.
  • Usar a opção 'Criare Continuar' significa que existe a necessidade de se cadastrar este projeto atual  e já partir para o cadastro de um novo. Útil não é? Imagine-se cadastrando dez projetos seguidos, isso já lhe economizaria alguns cliques.
