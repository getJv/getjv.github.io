# The qlik engine

Until now 2020 the qlik uses the old angularJS as their framework. Once we has access the qliksense we can use it to call each object from a qliksense folder, or each object from it, or we can create a custom object we our own configurations.

# Getting the qlik engine from qliksense desktop app

  1. Access your devhub environment. in my case: http://localhost:4848/dev-hub/;
  2. Select the option Create new mashup and fill out the project name and select your template (e.g.: teste-wiki, basic mashup template);

  1. Now the qlik will generete the common files:

    1. Now your app scaffold is done. you can edit then by qlik page or open all files in your prefered Editor/IDE.

    TIP

    • All extensions, mashups and qlik sites are under: [...sense path ...]\Sense\Extensions.
    • In my case I'm using the qlik sense desktop app so it is: C:\Users\YOUR-USERNAME\Documents\Qlik\Sense\Extensions
    • If you are using qlik from other server remember to enable CORS at QMC rules

    # Useful commands

    # Get the Current selection bar

      [...]
	var app = qlik.openApp('BancoHoras.qvf', config);

	//get objects -- inserted here --
	app.getObject('CurrentSelections','CurrentSelections');
[...]

      # Get values from a Generic object

        [...]
	 app = qlik.openApp("your-qvf-file.qvf");

     app.createGenericObject(
       {
         uf: { qStringExpression: "Count(Distinct UF)" },
       },
       function (reply) {
         console.log(reply.uf);
       }
     );
[...]

        TIP

        • When working with local qliksense app your qvf files usually are under: C:\Users\YOUR-USERNAME\Documents\Qlik\Sense\Apps
        • When you qvf file is published on qmc server it will receive another name ( a hashname )

        # Get object directly from a sheet

        Here we get the qlik object how-it-is and just reder it at html selected, QV01 in our sample.

      app.visualization.get("yNNBhN").then(function (viz) {
       viz.show("QV01");
     });

         },

             qLeft: 0,
